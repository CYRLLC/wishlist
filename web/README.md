# WishLink Web

這個資料夾是 WishLink 的 GitHub Pages 靜態 Web 版，保留原本 iOS App 的核心方向：帳號登入、情侶配對、願望卡、願望審核、打工點數與共同資金。

## 本機執行

```bash
cd web
npm install
npm run dev
```

沒有 Firebase 環境變數時會進入本機展示模式，資料只存在目前瀏覽器。

## Firebase 設定

### 1. 確認費用方案

MVP 可以先使用 Firebase Spark 免費方案，不需要綁信用卡。這個專案目前只需要：

- Authentication：Email/Password 登入
- Cloud Firestore：帳號、配對、願望、任務、點數、資金資料
- GitHub Pages：靜態網頁部署

避免開啟 Cloud Functions、Firebase App Hosting、Phone Auth、Google Cloud 付費 API，這些比較容易需要 Blaze 付費方案。

### 2. 啟用登入

在 Firebase Console：

1. Build -> Authentication。
2. 點 `Get started`。
3. Sign-in method -> Email/Password。
4. 啟用 `Email/Password`，不用啟用 email link。

### 3. 建立 Web App 並取得設定值

在 Firebase Console：

1. Project overview 旁邊的齒輪 -> Project settings。
2. General -> Your apps。
3. 點 `</>` Web app。
4. App nickname 可填 `WishLink Web`。
5. 不需要勾 Firebase Hosting，因為這裡使用 GitHub Pages。
6. 建立後複製 `firebaseConfig` 裡的值。

### 4. 本機填入環境變數

複製 `.env.example` 為 `.env.local`，填入 Firebase Web 設定。

需要的環境變數：

```bash
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
```

### 5. 套用 Firestore Rules

在 Firebase Console：

1. Build -> Firestore Database。
2. 進入 `規則` 分頁。
3. 將本專案的 `web/firestore.rules` 內容貼上。
4. 點發布。

### 6. 本機測試 Firebase 連線

```bash
cd web
npm install
npm run dev
```

如果 `.env.local` 有填對，畫面不會出現「本機展示模式」提示。可以先註冊兩個帳號，然後用其中一個帳號的邀請碼讓另一個帳號配對。

## 第一次使用流程

1. 使用者 A 註冊帳號，填 Email、密碼與暱稱。
2. 使用者 A 進入配對頁，複製自己的邀請碼給使用者 B。
3. 使用者 B 註冊或登入後，在配對頁輸入使用者 A 的邀請碼。
4. 配對完成後，首頁上方會顯示「我」與「對方」的資料。
5. 雙方可以開始新增願望、建立打工任務、留言與記錄共同資金。

建議測試時使用兩個瀏覽器、無痕視窗，或一台電腦加一支手機，分別登入兩個帳號。

## 功能區使用說明

### 登入與註冊

用途：建立個人帳號，讓資料能同步到 Firebase。

使用方式：

1. 第一次使用點 `還沒帳號？免費註冊`。
2. 輸入暱稱、Email、密碼。
3. 已有帳號時，切回登入模式輸入 Email 和密碼。

注意事項：

- 暱稱會顯示在雙方資料與留言板。
- 密碼至少 6 碼，這是 Firebase Email/Password 的基本限制。
- 如果登入失敗，先確認 Firebase Authentication 已啟用 Email/Password。

### 配對伴侶

用途：把兩個帳號綁定成同一組情侶，之後願望、任務、資金與留言都會共用同一個配對空間。

使用方式：

1. 登入後如果尚未配對，會進入配對頁。
2. 把自己的邀請碼複製給對方。
3. 對方登入後，把你的邀請碼貼到 `輸入對方邀請碼`。
4. 按 `開始配對`。

確認方式：

- 配對成功後會進入主畫面。
- 主畫面上方會顯示 `我` 與 `對方` 的暱稱和 Email。
- `我` 分頁會顯示雙方資料與 ID。

注意事項：

- 不能輸入自己的邀請碼。
- 如果顯示找不到邀請碼，請確認對方已經註冊成功。
- 如果剛配對後沒看到對方資料，重新整理或稍等 Firestore 同步。

### 願望區

用途：提出想要的物品、旅行、約會或其他願望，讓對方審核。

新增願望：

1. 進入 `願望` 分頁。
2. 點 `新增願望`。
3. 填入願望名稱、描述、說服區、渴望程度、緊急程度。
4. 可選填預估金額與購買連結。
5. 送出後，願望會出現在 `我的願望`。

審核對方願望：

1. 對方新增願望後，會出現在 `對方的願望`。
2. 可以選 `同意`、`駁回` 或 `暫緩`。
3. 駁回時可填駁回原因。
4. 暫緩時可設定需要多少點數才能兌換。

狀態說明：

- `待審核`：剛新增，等對方決定。
- `已承諾`：對方同意。
- `已駁回`：對方拒絕，會顯示原因。
- `暫緩集點`：需要累積指定點數後兌換。
- `已兌換`：已用點數兌換。
- `已完成`：保留給未來完成流程使用。

### 打工區

用途：設定可以換點數的任務，例如洗碗、按摩、陪跑步、整理房間。

建立任務：

1. 進入 `打工區`。
2. 輸入任務名稱。
3. 選擇週期：單次、每日、每週或每月。
4. 設定每次完成後可獲得的點數。
5. 點 `新增任務`。

申請點數：

1. 對方建立的任務會顯示為可申請。
2. 完成後可以填寫申請備註。
3. 點 `申請點數`。

確認給點：

1. 任務建立者會看到待確認的申請。
2. 確認無誤後點 `確認給點`。
3. 系統會把點數加到申請者帳號，並新增一筆點數紀錄。

注意事項：

- 建立任務的人負責審核。
- 點數會加給申請任務的人。
- 目前週期是用來標示任務頻率，不會自動建立下一輪任務；如果要完整自動重置，需要後端排程功能。
- 如果確認給點出現權限錯誤，請確認 Firestore Rules 已更新到本專案最新版。

### 留言板

用途：雙方可以在同一個配對空間留言，避免只有願望與任務但不知道對方想法。

使用方式：

1. 進入 `留言` 分頁。
2. 在輸入框寫下訊息。
3. 點 `送出留言`。
4. 雙方都會看到留言紀錄。

適合內容：

- 補充願望原因。
- 討論任務是否完成。
- 記錄約定、提醒與小紙條。
- 告訴對方目前想先完成哪個願望。

限制：

- 每則留言最多 500 字。
- 只有同一組配對中的兩個帳號可以讀取與新增留言。

### 點數區

用途：查看自己的點數、點數紀錄，以及兌換暫緩願望。

使用方式：

1. 進入 `點數` 分頁。
2. 上方會顯示目前可用點數。
3. `可兌換願望` 會列出自己的暫緩願望。
4. 點數足夠時，按 `兌換`。

規則：

- 點數屬於個人，不是共同點數。
- 點數不能轉讓給對方。
- 點數主要透過打工任務取得。
- 兌換願望後，會扣除對應點數並新增消費紀錄。

### 資金區

用途：記錄共同存款，並比較目前資金距離各願望還差多少。

新增存入：

1. 進入 `資金` 分頁。
2. 輸入存入金額。
3. 可填備註，例如本週存入、晚餐省下、薪水分配。
4. 點 `新增存入`。

願望資金比較：

- 有填預估金額的願望會出現在比較清單。
- 系統會顯示目前資金是否足夠。
- 如果不足，會顯示還差多少。
- 清單會依接近可負擔的程度排序。

注意事項：

- 資金區目前是手動記帳，不會真的串接銀行或付款。
- 存入紀錄是共同可見。
- 刪除或編輯存入紀錄目前尚未實作。

### 我 / 雙方資料

用途：確認目前登入者、邀請碼與配對對象，避免不知道跟誰配對。

可看到：

- 我的暱稱
- 我的 Email
- 我的邀請碼
- 對方暱稱
- 對方 Email
- 對方 ID

使用情境：

- 剛配對後確認對象是否正確。
- 要重新複製自己的邀請碼給對方。
- 發現資料不同步時，先到這裡確認目前登入帳號。

### 手機與電腦版差異

電腦版：

- 左側固定導覽列。
- 功能區可以雙欄顯示，例如我的願望與對方願望。

手機版：

- 導覽列會移到底部。
- 各區塊改成單欄，方便直向滑動。
- 建議用手機測試配對、留言與審核流程，確認實際使用情境。

## GitHub Pages 部署

`.github/workflows/deploy-pages.yml` 會在 `main` push 時建置 `web/` 並部署到 GitHub Pages。

在 GitHub repo 設定：

1. Settings -> Pages -> Source 選 GitHub Actions。
2. Settings -> Secrets and variables -> Actions，新增上方 `VITE_FIREBASE_*` secrets。
3. push 到 `main`。
